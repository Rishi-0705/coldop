    76→  - **Important deviation from spec (documented inline)**: socket.io path is `/socket.io` (default), NOT `/`. The websocket example uses path `/`, but that pattern only works for pure-socket.io processes. With path `/`, Engine.IO's attach middleware intercepts EVERY HTTP request (because every URL starts with `/`), which would block the `/broadcast` and `/health` REST routes on the same port. Caddy still routes correctly because it only inspects the `XTransformPort` query param, not the URL path. Browser client should call `io({ path: '/socket.io', query: { XTransformPort: 3003 } })`.
